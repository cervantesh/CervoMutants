# Upgrade Notes for v0.2.0

## Summary

- Report consumers should account for the new resource and structured-failure
  fields, including `memory_killed`, `skipped_resource`, `pending_budget`, and
  `stopped_reason`.

## Operator Action

- CI or parser code that assumed every failure surfaced only as a generic error
  should be updated to read the structured failure artifacts.

## Rollback

- Reinstall `v0.1.0` if downstream automation cannot yet consume the new
  structured failure fields, and keep older report consumers pointed at
  pre-`v0.2.0` artifacts until the parser upgrade is ready.

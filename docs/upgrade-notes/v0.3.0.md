# Upgrade Notes for v0.3.0

- Windows-native users should expect safer temp-root selection and conservative
  worker behavior during `temp-workdir` runs.
- Teams using survivor review heuristics should re-baseline any manual
  expectations around comparator-boundary ranking because calibration narrowed
  that noisy signal.
- Large-repo CI users can adopt slicing and sharding instead of relying on
  ad hoc bounded runs.
